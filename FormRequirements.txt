Notes for development

- Build homepage first, but keep it simple
- Then do the basic forms one by one
- Then do the complex forms
- Do not change too many files at once
- For foreign keys, use dropdowns if possible instead of asking user to type IDs
- Test each page right after finishing it

Testing notes

For each basic form:
- test add
- test view
- test update
- test delete

Keep screenshots for:
- before
- during
- after

For each complex form:
- test filter/search using dropdown or selection
- keep screenshot before selecting
- keep screenshot after result is shown

Also keep 2 or 3 error cases, for example:
- missing required field
- wrong or missing dropdown selection
- wrong date/time or invalid value

For each error case keep:
- bad input screenshot
- error screenshot
- corrected version screenshot
